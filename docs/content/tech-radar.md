# Tech Radar

The Nexus IDP Tech Radar is Stratpoint's technology adoption guide. It gives engineers a clear, shared view of which technologies are recommended, being evaluated, or being phased out.

---

## Reading the Radar

The radar is organized into four rings:

| Ring | Meaning |
|------|---------|
| **Adopt** | Proven, recommended for production use — this is our default choice |
| **Trial** | Worth using on a project with close attention — evaluate and share learnings |
| **Assess** | Interesting, worth exploring in a spike or side project — not ready for production |
| **Hold** | Not recommended for new projects — existing use should be migrated away |

Technologies are grouped into four quadrants:

| Quadrant | What it covers |
|----------|---------------|
| **Languages & Frameworks** | Programming languages, web/backend frameworks |
| **Platforms** | Cloud providers, container orchestration, CI/CD platforms |
| **Techniques** | Architectural patterns, development practices |
| **Tools** | Developer tooling, observability, security scanning |

---

## How to Use It

**Starting a new project?**
→ Check the radar before picking a stack. Default to **Adopt** ring technologies. For **Trial** ring, discuss with your tech lead first.

**Evaluating a new library or tool?**
→ Check if it's already on the radar. If it's in **Hold**, find out why before using it.

**Want to propose a change?**
→ Raise it in `#engineering-standards` on Slack. The radar is reviewed quarterly.

---

## Radar vs. Engineering Hub

The Tech Radar shows **what** to use. The [Engineering Hub](https://portal.stratpoint.io/engineering-docs) (Engineering Docs) shows **how** to use it — with detailed guides, golden paths, and examples.

---

## Accessing the Radar

Click **Tech Radar** in the left sidebar, or navigate to `/tech-radar`.

The radar is sourced from the [Thoughtworks Tech Radar](https://www.thoughtworks.com/radar) format and customized for Stratpoint's stack.
